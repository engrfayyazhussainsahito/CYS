Week 8 · Midterm Practical

Attack Surface and Defense Review

Week 8 is an integrated practical assessment. Students receive a fictional organization, architecture notes, code fragments, configurations, logs, and risk constraints. The task is to identify attack surface, prioritize risk, propose controls, and produce defensible evidence.

Estimated reading time90–150 minutes for instructions; 8–14 hours for the practical
Difficulty levelMS foundation integration
Required backgroundWeeks 0–7
DeliverablePractical report + evidence appendix + executive summary

Week Overview

This week does not introduce a new attack technique. It checks whether you can integrate previous concepts: asset identification, threat modeling, access control, authentication, cryptography, secure coding, web/API security, memory-safety thinking, network segmentation, detection, and reporting.

Learning Objectives

  • Build an asset and trust-boundary map for a fictional organization.
  • Identify realistic attack surface without performing unauthorized testing.
  • Distinguish vulnerability severity from organizational risk.
  • Prioritize remediation using likelihood, impact, exploitability, exposure, and compensating controls.
  • Translate technical findings into a management-ready defense plan.
  • Write concise evidence-backed security findings.

Required Background

Students should be able to read basic Linux commands, HTTP requests, Python snippets, file permissions, authentication logs, firewall-like rules, Mermaid diagrams, and simple web/API code. Every artifact is fictional and local to the course.
