Week 1

Security Mindset

Security mindset means learning to reason about value, harm, uncertainty, attackers, accidents, controls, risk, and professional responsibility before reaching for tools.

Estimated reading time12–18 hours
Difficulty levelBeginner → MS foundation
Required backgroundWeek 0 recommended
Major outputThreat model + ethics memo

Overview

This week introduces core security reasoning: assets, stakeholders, security objectives, harm, threats, vulnerabilities, controls, risk, threat modeling, legal authorization, professional ethics, and responsible disclosure.

Learning Objectives

  • Explain what cybersecurity protects and why those assets matter.
  • Define confidentiality, integrity, availability, privacy, safety, authenticity, accountability, and resilience.
  • Distinguish threats, vulnerabilities, exploits, controls, likelihood, impact, and risk.
  • Create a basic threat model for a small system.
  • Identify ethical and legal boundaries for security testing.
  • Separate observation, inference, uncertainty, and unsupported claims in a security report.

Required Background

No cybersecurity background is required. Week 0 provides useful preparation in Linux, networking, Python, Git, and lab documentation. Students who skipped Week 0 can still begin here, but technical labs later in the course will be harder.

Lessons

  1. Lesson 1.1: What Cybersecurity Protects ready
  2. Lesson 1.2: CIA Triad and Security Goals ready
  3. Lesson 1.3: Threats, Vulnerabilities, Exploits, and Risk ready
  4. Lesson 1.4: Threat Modeling from First Principles ready
  5. Lesson 1.5: Ethics, Law, and Responsible Disclosure ready

Week 1 Output

By the end of Week 1, students will produce a beginner-safe threat model for a fictional university portal and a short ethics memo explaining what testing is authorized, what is prohibited, and how uncertainty should be reported.
