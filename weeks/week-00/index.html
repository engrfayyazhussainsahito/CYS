Week 0 — Cybersecurity Bridge and Lab Readiness

This week creates the minimum technical foundation needed for safe cybersecurity learning. It does not assume previous Linux, networking, programming, or security experience.

Estimated time6–10 hours
DifficultyBeginner bridge
Required backgroundNone
OutcomeSafe, working lab environment

Overview

Students learn how to use an isolated lab, why authorization matters, and how Linux, networking, Python, Git, and virtual machines fit into cybersecurity practice.

Learning objectives

  • Explain safe lab boundaries.
  • Use basic Linux terminal commands.
  • Recognize IP addresses, ports, DNS, HTTP, and packets.
  • Run basic Python scripts.
  • Use Git for version history.
  • Create and restore VM snapshots.

Required background

No prior knowledge is required. The first lesson explains the ideas before asking students to use tools.
