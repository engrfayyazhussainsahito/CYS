Week 6

Web and API Security

Week 6 teaches the web from first principles before introducing web and API security, testing, and secure deployment. Students learn browsers, HTTP, cookies, sessions, same-origin policy, APIs, trust boundaries, common web vulnerabilities, secure design, and defensive testing in authorized labs.

Estimated reading time12–20 hours
Difficulty levelBeginner → MS foundation
Required backgroundWeeks 0–5 recommended
Major outputWeb architecture and safe security review report

Overview

Web systems are where users, browsers, identity, APIs, databases, cloud infrastructure, and business logic meet. This week starts with architecture and protocol foundations, then moves toward sessions, browser security, injection, XSS, CSRF, SSRF, API authorization, and secure deployment.

Learning Objectives

  • Explain browser/server architecture, HTTP, HTTPS, URLs, headers, cookies, sessions, APIs, and same-origin policy.
  • Identify trust boundaries in web applications and APIs.
  • Explain common web vulnerabilities and defenses at a conceptual level.
  • Use safe local labs to inspect requests, responses, cookies, forms, JSON, and logs.
  • Write a professional web security review focused on evidence and remediation.

Required Background

No previous web-development background is required. Students should understand basic networking, Python, identity, authorization, cryptography, and secure software concepts from earlier weeks. Each lesson reintroduces the prerequisites before using them.

Lessons

Lesson 6.1: Web Applications from First Principles

Browsers, URLs, DNS, HTTP, HTTPS, cookies, sessions, APIs, same-origin policy, CORS, web architecture, logs, and trust boundaries.

Lesson 6.2: Sessions, Cookies, and Browser Security

Session lifecycle, secure cookie attributes, browser storage, CSRF, same-origin policy, CORS, CSP, SRI, HSTS, and local-only labs.

Lesson 6.3: Injection, XSS, and Secure Output

Injection thinking, SQL injection concepts, XSS, context-aware output encoding, parameterized queries, safe DOM use, detection, and local-only labs.

Lesson 6.4: API Security and Object-Level Authorization

REST-style APIs, JSON, tokens, claims, scopes, tenants, BOLA/IDOR, object-property authorization, schemas, rate limiting, gateways, API logs, and safe local labs.

Lesson 6.5: Web Security Testing and Secure Deployment

Safe testing methodology, authorized scope, OWASP WSTG and ASVS thinking, DAST, secure headers, TLS, deployment gates, monitoring, rollback, and reporting.
