Week 4

Secure Software and Supply Chain

Week 4 teaches how software is planned, designed, coded, tested, built, released, and maintained securely. It connects secure coding with dependency risk, CI/CD, software bills of materials, artifact integrity, and secure-by-design thinking.

Estimated reading time12–20 hours
Difficulty levelBeginner → MS foundation
Required backgroundWeeks 0–3 recommended
Major outputSecure SDLC review and supply-chain risk memo

Overview

Software security is not only code scanning. It includes requirements, design, coding, testing, dependency selection, build integrity, deployment controls, monitoring, and patching. This week starts from first principles and gradually moves toward professional secure software programs and supply-chain assurance.

Learning Objectives

  • Explain software, source code, builds, releases, and deployments from first principles.
  • Write testable security requirements and abuse cases.
  • Apply secure design principles to small systems.
  • Recognize common vulnerability classes and secure coding mistakes.
  • Use defensive tools such as code review checklists, dependency inventories, and local secret-pattern checks.
  • Explain SBOMs, provenance, artifact signing, and CI/CD risk at an introductory graduate level.

Required Background

No software-engineering background is required. Students should understand basic Linux, Python, Git, threat modeling, identity, access control, and cryptographic concepts from earlier weeks. Each lesson re-explains the required prerequisite ideas.

Lessons

Lesson 4.1: Secure Software Engineering from First Principles

Builds the mental model: software lifecycle, requirements, design, code, tests, releases, operations, secure-by-design thinking, and safe local review.

Lesson 4.2: Requirements, Abuse Cases, and Secure Design

Security requirements, abuse cases, data flows, trust boundaries, design review, acceptance criteria, and abuse-driven tests.

Lesson 4.3: Input Validation, Output Encoding, and Error Handling

Defensive coding patterns for validating data entering software, encoding data leaving software, and handling errors safely.

Lesson 4.4: Secrets, Dependencies, and SBOMs

Secret lifecycle, dependency risk, direct and transitive components, SCA, SBOMs, VEX, provenance, and vulnerability prioritization.

Lesson 4.5: Testing, Fuzzing, CI/CD, and Release Integrity

Automated testing, fuzzing, CI/CD trust boundaries, pipeline security, artifact hashes, signing, provenance, release gates, and release evidence.
