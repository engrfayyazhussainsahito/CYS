Week 9 · Endpoint Security, Malware Concepts, and Hardening

Endpoint Security, Malware Concepts, and Hardening

Week 9 moves from networks into the devices people actually use: laptops, desktops, servers, mobile devices, and cloud-hosted workloads. The focus is safe defense: inventory, hardening, patching, logging, malware concepts, endpoint detection, and recovery.

Estimated reading time60–90 minutes for overview; 6–10 hours per full lesson
Difficulty levelBeginner → MS foundation
Required backgroundWeeks 0–8 recommended
DeliverableEndpoint inventory, hardening baseline, and patch review

Overview

An endpoint is a computing device or workload where users, applications, credentials, data, and attacker activity often meet. Endpoint security is not only antivirus. It combines asset inventory, secure configuration, least privilege, patching, application control, malware prevention, logging, detection, investigation, and recovery.

Learning Objectives

  • Define endpoints and explain why they are common attacker targets.
  • Describe the endpoint attack surface from hardware to user behavior.
  • Separate malware concepts from endpoint-defense controls.
  • Build a basic endpoint baseline for a safe lab system.
  • Explain patch management, update rings, exception handling, and configuration drift.
  • Identify useful endpoint telemetry for detection and investigation.
  • Explain how EDR alerts are triaged, tuned, and converted into authorized response actions.
  • Design ransomware readiness and recovery plans using backups, RTO/RPO, containment, and validation.

Required Background

Students should understand operating systems, users, permissions, services, logs, networks, authentication, and risk scoring. This week reviews prerequisite ideas before using them.

Week Lessons

  1. Lesson 9.1 — Endpoint Security from First Principles
  2. Lesson 9.2 — Malware Concepts and Safe Analysis
  3. Lesson 9.3 — Endpoint Hardening and Patch Management
  4. Lesson 9.4 — EDR, Telemetry, and Detection
  5. Lesson 9.5 — Ransomware Readiness and Endpoint Recovery

Safety Boundary

All activities are defensive and local. The labs use benign files, fictional logs, and normal operating-system tools. Students must not run real malware, bypass production security controls, remove security patches, or test systems they do not own or administer with written authorization.
