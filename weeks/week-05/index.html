Week 5

Memory Safety and Binary Foundations

Week 5 explains how programs occupy memory, why C and C++ can be powerful but dangerous, how memory-safety bugs occur, and how modern systems reduce the risk without assuming students already know computer architecture.

Estimated reading time12–20 hours
Difficulty levelBeginner → MS foundation
Required backgroundWeeks 0–4 recommended; no C background required
Major outputMemory-safety bug analysis and mitigation report

Overview

Many serious vulnerabilities come from programs mishandling memory. This week starts with the mental model of bytes, addresses, variables, pointers, stack, heap, and compiled binaries. It then connects those ideas to buffer overflows, use-after-free, integer errors, mitigations, sanitizers, fuzzing, memory-safe languages, and safe analysis methods.

Learning Objectives

  • Explain memory, bytes, addresses, stack, heap, and pointers from first principles.
  • Describe why C allows direct memory access and why that creates security risk.
  • Recognize common memory-safety bug classes without weaponizing them.
  • Use defensive tools such as compiler warnings and sanitizers in local labs.
  • Explain mitigations such as ASLR, DEP/NX, stack canaries, and memory-safe languages at a conceptual level.
  • Write professional vulnerability analysis focused on root cause and remediation.

Required Background

No C or assembly background is required. Students should understand basic Linux commands, Python, files, processes, secure software concepts, and local-lab safety from earlier weeks. Each lesson reintroduces the required prerequisite concepts before using them.

Lessons

Lesson 5.1: Memory, C, and Unsafe Code from First Principles

Bytes, addresses, variables, stack, heap, pointers, arrays, strings, unsafe C behavior, defensive compilation, and sanitizer-based local analysis.

Lesson 5.2: Stack, Heap, and Process Memory

Process address spaces, virtual memory, stack frames, heap allocation, memory mappings, permissions, and safe local inspection.

Lesson 5.3: Buffer Overflows and Memory Corruption

Out-of-bounds reads and writes, stack and heap overflows, off-by-one errors, sanitizer evidence, safe root-cause analysis, and defensive remediation.

Lesson 5.4: Sanitizers, Fuzzing, and Bug Triage

Sanitizers, fuzz targets, harnesses, seed corpora, mutation, coverage guidance, crash reproduction, minimization, deduplication, and evidence-based triage.

Lesson 5.5: Mitigations and Memory-Safe Languages

Stack canaries, ASLR, PIE, DEP/NX/W^X, RELRO, CFI, shadow stacks, sandboxing, memory-safe languages, FFI boundaries, and migration roadmaps.
