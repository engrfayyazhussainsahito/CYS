Week 10 · Cloud, Containers, and DevSecOps

Cloud, Containers, and DevSecOps

Week 10 moves from individual endpoints and networks into cloud-hosted systems. Students learn how cloud computing changes responsibility, identity, infrastructure, logging, deployment, and incident response.

Estimated reading time60–90 minutes for overview; 6–10 hours per full lesson
Difficulty levelBeginner → MS foundation
Required backgroundWeeks 0–9 recommended
DeliverableCloud architecture review and shared-responsibility map

Overview

Cloud security is the practice of protecting cloud-hosted data, identities, applications, infrastructure, logs, and deployment pipelines. Cloud systems are powerful because they are programmable and elastic. The same properties create risk: a single bad identity permission, public storage rule, leaked key, or unreviewed deployment script can expose large systems quickly.

Learning Objectives

  • Define cloud computing, service models, deployment models, and regions/zones.
  • Explain shared responsibility and why it changes across IaaS, PaaS, SaaS, serverless, and managed services.
  • Identify the basic cloud building blocks: compute, storage, network, IAM, APIs, logs, secrets, and key management.
  • Map cloud attack surfaces to preventive, detective, and recovery controls.
  • Explain why containers and DevSecOps need cloud-aware guardrails.
  • Prepare for later lessons on cloud IAM, containers, Kubernetes, infrastructure as code, and cloud incident response.

Required Background

Students should understand Linux, networking, identity, access control, cryptography basics, secure software, endpoint hardening, and logging. This week reviews cloud-specific versions of those ideas before introducing managed services and automation.

Week Lessons

  1. Lesson 10.1 — Cloud Security from First Principles
  2. Lesson 10.2 — Cloud IAM and Identity Governance
  3. Lesson 10.3 — Containers, Images, and Kubernetes Security
  4. Lesson 10.4 — DevSecOps Pipelines and Infrastructure as Code
  5. Lesson 10.5 — Cloud Logging, Monitoring, and Incident Response

Safety Boundary

All Week 10 activities are defensive. The first lesson uses fictional diagrams, local JSON files, and local scripts. Students must not scan cloud environments, access accounts they do not own, create public resources, mine cryptocurrency, or test production systems without explicit written authorization.
