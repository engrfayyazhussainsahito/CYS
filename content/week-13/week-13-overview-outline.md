# Week 13 Overview — Offensive Security Methodology

## Purpose
Teach authorized offensive-security methodology as disciplined risk validation, not unauthorized hacking.

## Research basis
- NIST SP 800-115 — planning and conducting technical information security tests and examinations.
- NIST SP 800-53 Rev. 5 — security/privacy control governance context.
- NIST SP 800-53A Rev. 5 — control assessment methodology.
- OWASP WSTG — web application testing structure.
- MITRE ATT&CK — common adversary-behavior vocabulary for purple-team planning.
- CISA Cyber Hygiene Services — authorized recurring vulnerability scanning example.
- CREST penetration testing guide — program management and testing governance guidance.
- PTES — industry phase model, supplemented by current legal, cloud, and organizational guidance.

## Lesson map
1. Lesson 13.1 — Authorized Offensive Security from First Principles.
2. Lesson 13.2 — Scoping, Rules of Engagement, and Test Planning.
3. Lesson 13.3 — Reconnaissance, Enumeration, and Attack Surface Mapping.
4. Lesson 13.4 — Vulnerability Validation, Exploit Safety, and Evidence.
5. Lesson 13.5 — Reporting, Remediation, Retesting, and Purple-Team Handoff.

## Safety boundary
Authorized, defensive, scoped, evidence-driven assessment only. No unauthorized exploitation, stealth, persistence, credential theft, destructive testing, evasion, or real-world target instructions.

## Deliverables
- Authorization and scope checklist.
- Rules-of-engagement document.
- Approved attack-surface map.
- Validation decision record.
- Finding template with remediation and retest criteria.
- Purple-team handoff checklist.
