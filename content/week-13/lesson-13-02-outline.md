# Lesson 13.2 — Scoping, Rules of Engagement, and Test Planning

## Purpose
Teach learners how to transform a vague offensive-security request into a bounded, authorized, safe, and useful assessment plan.

## Research anchors
- NIST SP 800-115 — technical security testing and assessment planning, execution, analysis, and mitigation.
- NIST SP 800-53A Rev. 5 — assessment planning, tailoring, risk tolerance, and analysis of assessment results.
- NIST SP 800-53 Rev. 5 — control context for authorization, monitoring, access control, audit, incident response, and risk management.
- OWASP WSTG — structured web and web-service security testing reference.
- MITRE ATT&CK — common adversary-behavior vocabulary for emulation and purple-team objectives.
- CISA Cyber Hygiene Services — authorized bounded vulnerability scanning model.

## Lab safety
The lab performs no scanning, exploitation, network activity, credential collection, or system interaction. It only evaluates fictional JSON plans for authorization and scope completeness.
