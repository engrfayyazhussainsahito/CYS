# Lesson 13.1 — Authorized Offensive Security from First Principles

## Purpose
Teach offensive security as authorized risk validation, not permissionless attack activity.

## Research anchors
- NIST SP 800-115 — testing and assessment planning, execution, findings analysis, mitigation.
- NIST SP 800-53 Rev. 5 and SP 800-53A Rev. 5 — control catalog and assessment context.
- OWASP WSTG — structured web application testing reference.
- MITRE ATT&CK — adversary-behavior vocabulary for defensive validation.
- CISA Cyber Hygiene — authorized vulnerability scanning as defensive exposure monitoring.
- PTES — useful older methodology model, supplemented by current legal/cloud/privacy guidance.

## Lab
Safe fictional Python decision aid that classifies assessment requests as ready, needing revision, or blocked. It performs no network, scanning, exploitation, credential, or external action.
