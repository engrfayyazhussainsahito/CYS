# Lesson 5.2 — Stack, Heap, and Process Memory

## Purpose
Teach process memory from first principles: virtual address spaces, pages, mappings, permissions, stack frames, call stacks, heap allocation, object lifetimes, memory leaks, dangling pointers, guard pages, ASLR, NX/DEP, and safe local process-map inspection.

## Safe Lab
Students compile a local C program, observe stack/heap/global/code addresses, inspect only their own process map, compare runs, and write a defensive analysis.

## Safety Boundary
No unauthorized process inspection, memory dumping, exploitation, or public-system testing.
