# Lesson 5.5 — Mitigations and Memory-Safe Languages

## Purpose
Teach layered memory-safety defense: prevention, detection, exploit mitigation, containment, and migration planning.

## Major sections
Stack canaries, ASLR, PIE, DEP/NX/W^X, RELRO, CFI, shadow stacks, pointer authentication, sandboxing, least privilege, memory-safe languages, garbage collection, Rust ownership, FFI boundaries, migration roadmap, safe local binary-hardening inspection lab.

## Safety
The lesson is defensive only. It uses local toy binaries and does not teach exploit development or unauthorized testing.
