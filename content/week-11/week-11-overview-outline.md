# Week 11 Overview — Security Operations, Monitoring, and Detection Engineering

## Purpose
Week 11 introduces security operations as an evidence-processing discipline. It connects telemetry, SIEM pipelines, detection engineering, triage, investigation, threat hunting, metrics, automation, and maturity.

## Core mental model
1. What evidence exists?
2. What behavior matters?
3. What decision is required?
4. What action is safe and authorized?

## Planned lesson sequence
- Lesson 11.1 — Security Operations from First Principles
- Lesson 11.2 — SIEM, Log Sources, and Telemetry Pipelines
- Lesson 11.3 — Detection Engineering and Rule Lifecycle
- Lesson 11.4 — Alert Triage, Investigation, and Threat Hunting
- Lesson 11.5 — Metrics, Automation, and SOC Maturity

## Practical deliverables
- SOC workflow map
- Telemetry inventory
- Detection brief
- Triage note
- Metrics review

## Safety boundary
Use local fictional data only unless explicit authorization exists. Do not collect or inspect real user, company, cloud, endpoint, email, or network telemetry outside approved scope.
