Website Architecture

This scaffold is designed for free GitHub Pages hosting, low maintenance, accessibility, and gradual lesson-by-lesson growth.

1. Design Constraints

  • Free hosting: no server, database, paid build service, or backend.
  • Beginner maintainability: plain HTML, CSS, and JavaScript instead of a complex framework.
  • Modular expansion: each week and lesson is a separate page.
  • Learning features: search, diagrams, quizzes, bookmarking, progress tracking, and print-to-PDF are implemented client-side.
  • Safety: all labs must be isolated and authorized.

4. Page Template

Every lesson page uses the same structure: metadata, breadcrumbs, toolbar, table of contents, content sections, self-assessment, lab, further reading, and previous/next navigation.

6. Content Generation Rule

New content should be added one lesson at a time. Each lesson must define all technical terms before using them, include safe labs only, and avoid instructions for unauthorized real-world activity.

7. Deployment

  1. Create a GitHub repository.
  2. Upload these files to the repository root.
  3. Open repository settings and enable GitHub Pages from the branch/root publishing source.
  4. Visit the published URL after GitHub Pages finishes deployment.
